# First Customer Demo

## Goal
Design the exact demo flow that will help sell V1 to the first paying user.

## Demo Audience
- freelance consultant
- solo analyst
- small agency doing recurring client reporting

## Demo Promise
Show how the product turns messy client spreadsheets into trustworthy, client-ready reporting output fast.

## Demo Flow

### 1. Start With The Pain
Show a messy client CSV/XLSX file.
Explain:
- ugly structure
- unclear headers
- missing values
- duplicates
- hard to compare
- hard to turn into a polished report quickly

### 2. Upload
Show:
- file upload
- automatic intake
- structure detection
- warnings

### 3. Cleaning & Health
Show:
- cleaning actions
- health score
- suspicious columns
- key data quality warnings

### 4. Insights
Show:
- top findings
- best charts
- executive summary draft
- why the findings matter

### 5. Compare
Show:
- current vs previous file
- biggest changes
- schema changes
- what should be mentioned to the client

### 6. Report
Show:
- selected findings
- selected charts
- summary blocks
- comparison summary
- report preview

### 7. Export
Show:
- polished export
- client-ready feel
- fast turnaround

## What The Demo Must Prove
- fast time to value
- handles messy files
- builds trust
- saves time
- improves client delivery quality

## What The Demo Must Avoid
- too many advanced features
- too much technical detail
- generic "AI magic" language
- showing weak or low-trust outputs

## Main Demo Message
This is not just analysis.
This is a client delivery workflow.

## Demo Asset Checklist

### Files

- [ ] **Sample messy file (Month 1)**
  Realistic CSV or XLSX with: 3–5 row preamble, mixed currency/percentage/date formats, 8–15% missing values, a handful of duplicate rows, at least one column with >50% nulls, inconsistent header casing (e.g. `Total Rev.` vs `total_revenue`)
  Suggested domain: marketing spend report, sales pipeline, or ops KPI sheet — common in consultant work

- [ ] **Sample previous-month file (Month 0)**
  Similar schema to Month 1 but with: slightly different column names (to show schema diff), lower row count, one column removed, one added, meaningfully different metric values (to show the comparison is useful)

- [ ] **Sample finished report (PDF)**
  A pre-generated PDF export from the product using the demo data. Must look polished enough to send to a client. Shows the end state before the demo starts — "this is what you'll have at the end."

- [ ] **Sample finished report (XLSX)**
  Same analysis exported as Excel — cover sheet, insights tab, column profiles tab, cleaning log tab, data preview tab. For audiences who prefer Excel deliverables.

- [ ] **Branded example project**
  A project already set up in the app with a plausible client name (e.g. "Acme Retail — April Report"), so the demo opens on something that looks real rather than "Test Project 1"

### Visuals

- [ ] **Before/after screenshot**
  Side-by-side: the raw messy CSV on the left, the finished polished report on the right. Used as the opener or the hero image.

- [ ] **Health score screenshot**
  A clean screenshot of the data health view showing a meaningful score (not 100, not 0) with visible cleaning actions applied

- [ ] **Insights screenshot**
  Top 3–5 findings with severity badges and evidence, no weak or trivial findings visible

- [ ] **Compare screenshot**
  Schema diff view and metric delta view side by side — schema pills showing 2 columns added, 1 removed, and a clear metric change in amber

- [ ] **Report builder screenshot**
  Template picker active, insights selected, export buttons visible — shows the assembly step

- [ ] **Live flow or screen recording**
  A 2–3 minute recorded run-through of the full demo flow. Used for async demos (email follow-ups, product page). No voice required — on-screen annotations are enough.

### Pricing

- [ ] **Pricing one-liner**
  A single sentence that explains the pricing without the user needing to visit the pricing page.
  Draft: "Free to try — Consultant plan at $39/month unlocks exports, comparison, and AI summaries."

- [ ] **ROI framing statement**
  One sentence that connects the price to saved time.
  Draft: "If this saves you 2 hours on one client report per month, it pays for itself in the first use."

- [ ] **Objection answer: "Can't I just do this in Excel?"**
  A short, honest answer ready to go.
  Draft: "You can, but you'd spend 1–2 hours cleaning, formatting, and writing. This does that in minutes, and the output looks more polished than most Excel reports."

### Readiness Checks

- [ ] The messy file uploads without errors
- [ ] The analysis runs in under 30 seconds on the demo file
- [ ] The health score shows something meaningful (not perfect, not broken)
- [ ] At least 3 strong insights appear — no trivial or confusing findings visible
- [ ] The compare step runs cleanly with the two demo files and shows visible differences
- [ ] The PDF export opens correctly and looks client-safe
- [ ] The XLSX export opens in Excel without errors
- [ ] The demo project can be reset quickly (or a new one created in under 1 minute) for back-to-back demos
